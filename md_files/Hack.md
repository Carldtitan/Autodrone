## **1️⃣ Your Goal: Hackathon Themes**

**These are the required themes. Every project must build in one of these three themes.**

- **Continual Learning:** How can LLM systems continuously improve from real-world use? Build agents, harnesses, or frameworks that allow continual learning through memory, user feedback, prompt optimization, self-reflection, toolkit expansion, or other methods. Focus on techniques that adapt in production, becoming more useful the more they are used with as little user intervention as possible


- **The Self-Improvement Stack:** What infrastructure is needed to continuously evaluate, monitor, and upgrade AI systems? Build the underlying tools that make self-improvement possible at scale, whether it be evaluation frameworks, novel observability systems, deployment loops, model routing, or automated experimentation platforms.


- **Recursive Intelligence:** This is what happens when models start to build themselves. We want you to facilitate RSI, building tools allowing models to directly improve their raw weights, whether through generating pretraining data, optimizing their hyperparameters, updating their architecture, and iterating on the best training techniques over time. Explore early forms of recursive or self-directed intelligence.

**🏆 Special Prizes (regardless, your project must build in one of the three above themes):**

- **Best Usage of DigitalOcean:** DigitalOcean credits awarded to the winning team.
- **Best Usage of LiveKit:** Keychron Q3 Max keyboards for the winning team.
- **Best Usage of Gemini 3.5:** $5000 Cash

  RSI can’t be static. The AI landscape is shifting to active, real-time agents. Your challenge this weekend is to build an application, tool, or workflow that leverages the absolute latest, bleeding-edge releases of the Gemini 3.5 ecosystem.

  We don't want to see standard wrapper chatbots or basic prompts. **We want to see something completely unprecedented.** Harness the power of autonomous agent environments, real-time multi-modal streaming, and visual computer automation to build the future of agentic workflows: the ones that improve themselves over time.
  - **🛠️ The Required Tech Stack**

    To qualify for this prize, your project **must** incorporate at least one (bonus points for combining multiple) of the following new features:
    - **🚀  Managed Agents in the Gemini API** 

      Stop managing complex local orchestration loops, scaffolding, or isolated local runtime containers. Leverage the **Interactions API** to call Google's hosted agent infrastructure (the **Antigravity agent** (antigravity-preview-05-2026).
      - **What it does:** With a single API call, spin up an autonomous agent that reasons, browses the web, and executes code natively inside an isolated, ephemeral Google-hosted Linux environment.

      - **Stateful Memory:** Pass the environment ID in follow-up calls to resume the session with all files, code, and terminal state completely intact.

      - **Define Your Skills:** Declare your agent's persona and custom skills cleanly using local [AGENTS.md](http://AGENTS.md) and [SKILL.md](http://SKILL.md) markdown files 
    - **🖥️  Computer Use in Gemini 3.5 Flash**

      Give your tool hands and eyes. Computer use is now natively integrated directly into the **Gemini 3.5 Flash** model.
      - **What it does:** The model can look at screens via screenshots and interact directly by generating native UI actions (mouse clicks, drags, keystrokes) across browser, desktop, and mobile environments.

      - **Use Cases:** Automate complex knowledge work across apps, build continuous software testers, or let an agent navigate the web on your behalf 
    - **🎙️ Live Translate via the Gemini Live API**

      Break down language barriers with zero-friction, continuous audio translation
      - **What it does:** Powered by the gemini-3.5-live-translate-preview model, the **Gemini Live API** supports low-latency, real-time speech-to-speech translation between 70+ languages.

      - **How it works:** Unlike classic turn-by-turn systems that wait for a speaker to finish, Live Translate uses a continuous WebSockets stream to translate seamlessly just a few seconds behind the speaker without awkward pauses .
    - **🍌 On-device & GenMedia** 

      Want to build a hyper-creative or fully offline agent? You are highly encouraged to mix in:
      - **Nano Banana:** Google's high-speed visual generation models, perfect for lightning-fast visual creation and precise text-in-image rendering. You can also explore Veo for video generation and Lyria for music generation.  

      - **Gemma 4:** Bring highly optimized, lightweight reasoning directly on-device or to the edge.

## **8️⃣ Judging Process**

**Round One:** Hackers will be assigned to judging groups in different rooms of the venue. Each team will have \~3 minutes to live demo their project, followed by 1-2 minutes of Q&A. The following criteria will be used:

- **Technicality (40%):** How difficult is this project to recreate? Does this involve technicality beyond your average hackathon project? Are you impressed with what they built in such a short time?


- **Live Demo (20%):** How is the actual demo of the project? Does it impress you? Is it well-engineered and working?


- **Creativity and Originality (25%):** How creative is the project? Is this something you have never seen before?


- **Future Potential and AI Impact (15%):** Does this project point toward an interesting future for AI? Does it advance how we think about AI capabilities, autonomy, learning, evaluation, or agentic systems?

**Round Two:** The top six teams demoing on stage in front of a panel of judges & all attendees. Each team will have \~3 minutes to live demo their project, followed by 1-2 minutes of Q&A. The same criteria as above will be used, though with equal weighting for each category.
